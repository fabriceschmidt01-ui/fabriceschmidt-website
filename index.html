Case Study: FANA Property Management

Das Problem

FANA Property Management betreibt Arbeiterunterkünfte in München.

  • Täglich: Buchungsanfragen per Email
  • Multilingual: Deutsch, Englisch, Polnisch, Rumänisch
  • Problem: Repetitiv, zeitintensiv, bindet Ressourcen

Kann AI das lösen?

Das Experiment

250 Stunden Solo-Entwicklung.

6-Agenten System gebaut:

Portal-Checker (welches Portal?)
Email-Extraktor (Daten rausziehen)
Sentiment-Analyse (CinnaMon)
Preis-Check (Hatch)
Response Generator (Sugar)
Quality Check (Bishop)

+ Human-in-the-Loop via Slack

Das Ergebnis

  • ✓ Funktioniert
  • ✓ Multilingual
  • ✓ Quality Check integriert
  • ✓ Produktionsreif

Die Learnings

  • Testing = 20% des Projekts (unterschätz das nicht!)
  • Multi-Agent robuster als Single-LLM
  • Human-in-the-Loop nicht optional
  • Es funktioniert wirklich - und das Problem gilt für tausende Firmen
